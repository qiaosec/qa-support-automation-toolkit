# 项目说明（中文）

## 这个项目是什么

这是一个偏 QA、技术支持、自动化方向的 Python 项目。

它可以读取工单导出数据，完成：

- 工单优先级标准化
- 风险评分
- SLA 风险识别
- 重复问题检测
- 回归测试候选项总结
- Markdown / JSON 报告输出

## 这个项目为什么适合放到 GitHub

因为它不是单纯的练手代码，而是一个能体现岗位相关性的项目：

- 如果投 QA：可以强调测试思维、缺陷归类、回归候选识别
- 如果投技术支持：可以强调工单分流、优先级判断、SLA 风险意识
- 如果投自动化方向：可以强调结构化输入、自动处理、自动报告输出

## 你可以怎么介绍它

你可以把它介绍成：

“我做了一个支持与 QA 自动化工具，可以分析工单数据、识别高风险问题、检测重复故障，并自动生成回归测试候选清单。这个项目展示了我在 QA、技术支持和自动化流程上的思考方式。”
