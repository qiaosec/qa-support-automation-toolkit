# Portfolio Pitch (English)

## Short version

I built a Python support-operations toolkit that analyzes ticket exports, scores urgency, identifies SLA risk, detects duplicate issues, and generates triage-friendly reports for QA and support teams.

## Resume version

Built a Python toolkit for QA and technical support workflows that ingests CSV or JSON ticket exports, normalizes issue data, ranks incidents by urgency, detects likely duplicates, and produces Markdown/JSON reports with regression checklist candidates.

## LinkedIn / recruiter version

This project demonstrates how I think about QA and support automation in a practical way. Instead of building a generic demo app, I built a tool that processes support ticket data, prioritizes operational risk, groups duplicate issues, and surfaces regression candidates for QA follow-up.

## Interview framing

If asked about the project, you can explain it like this:

1. I wanted a project that connected QA thinking, support workflows, and automation.
2. I designed a CLI tool that reads support ticket exports and turns them into actionable reports.
3. The project shows how I think about triage, duplicate problems, SLA risk, and repeatable reporting.
4. I added tests and documentation so it is not just code, but a maintainable and reviewable project.
